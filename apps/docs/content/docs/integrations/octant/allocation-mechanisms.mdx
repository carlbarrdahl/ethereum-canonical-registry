# Allocation Mechanisms

Curator Studio uses curator-defined weights. Octant V2 provides modular allocation mechanisms (quadratic funding, quadratic voting). These can combine.

## Hybrid Allocation

```
YieldRedirector → Strategy
                    │
         ┌──────────┴──────────┐
         │                     │
    80% Curator           20% Community
    (weighted)            (Octant QV round)
```

## Technical Integration

**Option A: Octant contract as allocation recipient**

```solidity
// Curator Strategy allocations
Allocation[] allocations = [
    { recipient: protocolGuild, weight: 40, label: "Protocol Guild" },
    { recipient: clientTeams, weight: 40, label: "Client Teams" },
    { recipient: octantQVContract, weight: 20, label: "Community Round" }
];
```

The Octant QV contract receives 20% and runs its own allocation round to sub-distribute.

**Option B: Octant determines full allocation**

```solidity
// Strategy defers to Octant allocation result
function distributeViaOctant(address octantRound) external {
    // Query Octant round results
    (address[] memory recipients, uint256[] memory amounts) = 
        IOctantRound(octantRound).getAllocationResults();
    
    // Distribute according to community decision
    warehouse.batchDeposit(recipients, token, amounts);
}
```

## Why This Works

- Curators provide baseline allocation (expertise)
- Community provides input on portion (legitimacy)
- Octant's battle-tested QF/QV mechanisms avoid reimplementation
- Composable: swap allocation mechanisms without changing Strategy

## Considerations

- Round timing: Octant rounds are periodic; Curator flows are continuous
- Eligibility: Who can participate in the Octant round?
- Weight of community input: configurable per Strategy
