# Shared Vaults

Both protocols wrap ERC-4626 vaults. Rather than parallel implementations, use Octant's audited vaults as yield source and Curator's allocation layer for distribution.

## Architecture

```
┌─────────────────────────────────────┐
│      Octant Funding Vault           │
│  - Yearn v3 audited patterns        │
│  - Multi-strategy support           │
│  - Risk management                  │
└─────────────────┬───────────────────┘
                  │ yield
                  ▼
┌─────────────────────────────────────┐
│     Curator Studio Strategy         │
│  - Curator-defined allocations      │
│  - Fund-of-funds composability      │
│  - Reputation/accountability        │
└─────────────────────────────────────┘
```

## Technical Integration

**Deprecate YieldRedirector4626**: Instead of maintaining a separate vault wrapper, point capital providers to Octant Funding Vaults.

```typescript
// Instead of:
await sdk.yieldRedirector.create({
  sourceVault: morphoVault,
  yieldRecipient: strategy,
});

// Use:
// 1. Deposit to Octant Funding Vault
// 2. Set donation address to Curator Strategy
await octantVault.deposit(amount, depositor);
await octantVault.setDonationAddress(strategy);
```

**Curator receives yield**: Same as direct funding—tokens arrive, `distribute()` routes them.

## Why This Works

- Single audited vault implementation (Octant)
- Curator focuses on allocation logic, not vault security
- Capital providers get Octant's risk management + Curator's allocation
- Reduces duplicate security audit costs

## Considerations

- Dependency: Curator depends on Octant vault availability/uptime
- Governance: Octant vault upgrades could affect Curator flows
- Branding: Clear attribution for both protocols' contributions
