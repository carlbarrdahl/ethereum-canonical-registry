# Strategy as Recipient

Octant Funding Vaults route yield to a configurable "donation address." A Curator Studio Strategy can serve as this recipient.

## Flow

```
Octant Funding Vault (USDC)
        │
   yield harvest
        │
        ▼
Curator Studio Strategy
├── 40% → Protocol Guild
├── 35% → Client Teams  
└── 25% → Developer Tools
        │
   distribute()
        │
        ▼
   SplitsWarehouse → Recipients
```

## Technical Integration

**Octant side**: Set `donationAddress` to a Curator Strategy address.

```solidity
// Octant vault configuration
vault.setDonationAddress(curatorStrategyAddress);
```

**Curator side**: Strategy receives tokens like any other funding source. No changes needed.

```solidity
// On Octant harvest, tokens arrive at Strategy
// Anyone calls distribute() to route to recipients
strategy.distribute(USDC);
```

## Why This Works

- Octant handles yield generation, risk management, audited vault infrastructure
- Curator handles opinionated sub-allocation with curator accountability
- Recipients claim from SplitsWarehouse as normal
- Full transparency: Octant harvest events + Curator distribution events

## Considerations

- Curator fees appear as allocations—visible to Octant depositors
- Distribution timing: manual, keeper-triggered, or on Octant harvest callback
- Attribution: index both Octant source and Curator strategy for provenance
