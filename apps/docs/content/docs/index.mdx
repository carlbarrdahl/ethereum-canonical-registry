# Canonical Registry

The **Canonical Registry** maps off-chain identifiers (GitHub repositories, DNS domains) to Ethereum addresses. Anyone can send ERC-20 tokens to a deterministic deposit address for any identifier — even before the owner has claimed it. When the owner proves ownership, they can withdraw accumulated funds.

## How It Works

```
  Anyone computes deposit address (no RPC call needed)
             │
             ▼
  ┌─────────────────────────────────┐
  │   github:org/repo               │
  │   0x4a3b...  (ClaimableEscrow)  │◀── Funders send tokens here
  └─────────────────────────────────┘
             │
  Owner proves GitHub/DNS ownership
             │
             ▼
  CanonicalRegistry records owner
             │
             ▼
  withdraw(token) → funds sent to owner
```

1. **Compute**: Anyone derives the deterministic deposit address for any identifier — pure local calculation, no wallet needed
2. **Fund**: Send ERC-20 tokens directly to the deposit address at any time — before or after claiming
3. **Claim**: Owner submits a verifiable proof (signed by the oracle backend after GitHub OAuth or DNS TXT check)
4. **Withdraw**: Anyone calls `withdraw(token)` — funds always go to the registered owner

## Value Proposition

| Stakeholder | Value |
| --- | --- |
| **Funders** | Send tokens to any open-source project or domain today, even if the owner hasn't set up a wallet yet |
| **Maintainers** | Claim accumulated funds any time after proving ownership — no setup required in advance |
| **Ecosystem** | Canonical, collision-free namespace for any off-chain identity on Ethereum |

## Getting Started

### SDK Setup

```typescript
import { CanonicalRegistrySDK } from "@ethereum-canonical-registry/sdk";

const sdk = new CanonicalRegistrySDK(walletClient);
```

Supported chains: `31337` (Hardhat), `11155111` (Sepolia), `1` (Mainnet — planned).

### Resolve an Identifier

```typescript
const state = await sdk.registry.resolveIdentifier("github", "org/repo", tokenAddress);
// state.id             — bytes32 identifier
// state.depositAddress — where funders send tokens
// state.owner          — null if unclaimed
// state.balance        — current ERC-20 balance at deposit address
```

### Fund an Identifier

```typescript
await tokenContract.transfer(state.depositAddress, amount);
```

### Claim and Withdraw

```typescript
// 1. Claim with a proof from the oracle signing service
await sdk.registry.claim("github", "org/repo", proofBytes);

// 2. Withdraw accumulated funds to the registered owner
await sdk.escrow.withdraw(state.depositAddress, tokenAddress);
```

### Parse URLs

```typescript
import { parseUrl } from "@ethereum-canonical-registry/sdk";

parseUrl("https://github.com/org/repo");
// → { namespace: "github", canonicalString: "org/repo", formatted: "github:org/repo" }

parseUrl("example.com");
// → { namespace: "dns", canonicalString: "example.com", formatted: "dns:example.com" }
```

### React Hooks

Wrap your app with `CanonicalRegistryProvider`, then use hooks:

```tsx
import {
  CanonicalRegistryProvider,
  useIdentifier,
  useClaim,
} from "@ethereum-canonical-registry/sdk";

function App() {
  return <CanonicalRegistryProvider>{/* ... */}</CanonicalRegistryProvider>;
}

function IdentifierCard({ id }: { id: `0x${string}` }) {
  const { data } = useIdentifier(id);
  const claim = useClaim();

  return (
    <div>
      <p>Owner: {data?.owner ?? "Unclaimed"}</p>
      <p>Deposit: {data?.escrowAddress}</p>
    </div>
  );
}
```
